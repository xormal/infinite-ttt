# Как пушить изменения в Git

Эти короткие шаги помогут быстро закоммитить и отправить изменения в удалённый репозиторий.

1. **Проверьте статус** – убедитесь, какие файлы изменены:
   ```bash
   git status --porcelain
   ```

2. **Добавьте файлы** в индекс (можно добавить конкретный файл или все изменения):
   ```bash
   git add <path/to/file>   # один файл
   # или
   git add .                # все изменения в текущей директории
   ```

3. **Создайте коммит** с понятным сообщением:
   ```bash
   git commit -m "Краткое описание изменения"
   ```

4. **Отправьте** коммит в удалённый репозиторий (обычно `origin` и ветка `main`):
   ```bash
   git push origin main
   ```

5. **Проверьте** результат:
   ```bash
   git log --oneline -5   # последние 5 коммитов
   ```

**Полезные подсказки**

- Если пуш отклонён из‑за конфликтов, сначала выполните `git pull --rebase`, решите конфликты и повторите `git push`.
- При работе в ограниченной среде (песочнице) может потребоваться запрос прав на запись в репозиторий. В таком случае используйте `with_escalated_permissions` в соответствующих командах.

Запомните эти шаги, и пуш будет проходить без лишних вопросов.
